## 3.2.2 骨干网络（Backbone）

YOLOv8 的骨干网络采用 CSPDarknet 作为特征提取器，由一系列卷积模块和 C2f 模块交替组成，负责从输入图像中逐层提取多尺度特征。

### 1. Focus 模块（空间下采样）

Focus 模块是 YOLOv8 对输入图像进行空间降采样的关键组件，其核心思想是通过**像素切片（Pixel Space Sampling）**操作将特征图的空间维度减半，同时通道数扩展至原来的 4 倍。

**数学定义：**

对于输入特征图 $X \in \mathbb{R}^{H \times W \times C}$，Focus 模块将其空间下采样 $2\times$，输出 $Y \in \mathbb{R}^{\frac{H}{2} \times \frac{W}{2} \times 4C}$：

$$Y_{i,j,k} = X_{\text{row}(k), \text{col}(k)}$$

其中：
- $\text{row}(k) = \lfloor k / 4 \rfloor \mod H$  
- $\text{col}(k) = k \mod W$

 Focus 模块不包含可学习参数，仅进行通道重组操作，计算量为零。

### 2. C2f 模块（CSP Bottleneck with 2 Convolutions）

C2f 模块是 YOLOv8 的核心构建块，源自 YOLOv5 的 C3 模块并进行了改进。其设计融合了**跨阶段局部网络（CSPNet）**的思想，通过特征分离和密集连接增强梯度流动。

**数学定义：**

给定输入特征 $x \in \mathbb{R}^{H \times W \times C_{in}}$，C2f 模块的计算过程如下：

$$
\begin{aligned}
x_1 &= \text{Conv}_1(x) \\
x_2 &= \text{Split}(x_1) \\
x_3 &= \text{Bottleneck}_1(x_2) \\
x_4 &= \text{Bottleneck}_2(x_3) \\
x_5 &= \text{Bottleneck}_3(x_4) \\
\ldots \\
x_{n+2} &= \text{Bottleneck}_n(x_{n+1}) \\
y &= \text{Conv}_2(\text{Concat}(x_2, x_3, x_4, \ldots, x_{n+2}))
\end{aligned}
$$

其中：
- $\text{Conv}_1$：$1\times1$ 卷积，用于通道分离
- $\text{Split}$：通道分割操作，将特征分为两部分
- $\text{Bottleneck}$：瓶颈层，由 $1\times1$ 卷积 + $3\times3$ 卷积 + 残差连接组成
- $\text{Concat}$：通道维度拼接
- $\text{Conv}_2$：$1\times1$ 卷积，用于通道融合

**C2f 模块结构图：**

```
┌────────────────────────────────────────────────────────────────┐
│                         C2f Module                              │
├────────────────────────────────────────────────────────────────┤
│                                                                    │
│         x ──→ [Conv1×1] ──→ ┬──→ [Bottleneck₁] ──→ ┐           │
│                              │                        │           │
│                              │──→ [Bottleneck₂] ──→ Concat ──→    │
│                              │                        │    [Conv1×1] ──→ y
│                              │──→ [Bottleneck₃] ──→ ┘           │
│                              │                                   │
│                              └─── skip ───────────────────────→ ┘
│                                                                    │
└────────────────────────────────────────────────────────────────┘
```

### 3. SPPF 模块（Spatial Pyramid Pooling - Fast）

SPPF 模块通过多尺度最大池化操作实现空间金字塔池化，有效融合不同感受野的特征，增强模型对目标尺度变化的鲁棒性。

**数学定义：**

对于输入特征 $x$，SPPF 模块的计算过程为：

$$
y = \text{Concat}\left(
    \text{MaxPool}(x, k=5),
    \text{MaxPool}(x_1, k=5),
    \text{MaxPool}(x_2, k=5),
    x
\right)
$$

其中 $x_1 = \text{MaxPool}(x, k=5)$，$x_2 = \text{MaxPool}(x_1, k=5)$。

**SPPF 与 SPP 的区别：**

| 特性 | SPP | SPPF |
|------|-----|------|
| 池化方式 | 并行 $5\times5, 9\times9, 13\times13$ | 串行 $5\times5$ MaxPool × 3 |
| 计算效率 | 较低 | 更高（减少约 50% 计算量） |
| 感受野 | $5\times5, 9\times9, 13\times13$ | $5\times5, 9\times9, 13\times13$（等效） |

## 3.2.3 颈部网络（Neck）

YOLOv8 的颈部网络采用**路径聚合网络（PANet）**结合**特征金字塔网络（FPN）**的双向特征融合架构，实现多尺度特征的充分交互。

### PANet 双向特征融合

PANet 在 FPN 自顶向下融合的基础上，额外增加了自底向上的特征增强路径，形成双向特征流动。

**数学定义：**

设 $P_i$ 表示 FPN 第 $i$ 层的特征，$N_i$ 表示 PAN 第 $i$ 层的增强特征：

**自顶向下融合（FPN）：**
$$
P_i = \text{Conv}\left(\text{Upsample}(P_{i-1})\right) \oplus \text{Conv}(C_i)
$$

其中 $C_i$ 是骨干网络第 $i$ 层的输出，$\oplus$ 表示通道拼接。

**自底向上增强（PAN）：**
$$
N_i = \text{Conv}\left(\text{Concat}\left(
    P_i,
    \text{Conv}(\text{Downsample}(N_{i-1}))
\right)\right)
$$

### 特征融合示意

```
                    ┌─────────────────────────────────────────┐
                    │              FPN (Top-down)               │
                    │                                          │
         C5 ──→ [Conv] ─────────────────────────────┐         │
                    │                               │         │
         C4 ──→ [Conv] ──→ [Upsample 2×] ──→ [Concat] ──→ P4│
                                                  ↓           │
         C3 ──→ [Conv] ──→ [Upsample 2×] ──→ [Concat] ──→ P3│
                                                  ↓           │
                    └─────────────────────────────────────────┘
                                                   │
                    ┌─────────────────────────────────────────┐
                    │              PAN (Bottom-up)             │
                    │                                          │
         P3 ──→ [Concat] ──→ [C2f] ──→ [Downsample] ──→ N3 ──┤
                                                  ↓           │
         P4 ──→ [Concat] ──→ [C2f] ──→ [Downsample] ──→ N4 ──┤
                                                  ↓           │
         P5 ──→ [Concat] ──→ [C2f] ───────────────────────→ N5│
                    └─────────────────────────────────────────┘
```

## 3.2.4 检测头（Head）

YOLOv8 采用**无锚框解耦检测头（Anchor-free Decoupled Head）**，将分类和回归任务解耦为两个独立的分支。

### 1. 解耦头设计

传统 YOLO 系列使用耦合检测头，共享特征同时预测分类和边界框。YOLOv8 将两个任务完全解耦，使用独立的分支进行预测。

**数学定义：**

对于输入特征 $F \in \mathbb{R}^{H \times W \times C}$：

**分类分支：**
$$
\begin{aligned}
f_{cls} &= \text{Conv}_{3\times3}(F) \\
f_{cls} &= \text{Conv}_{1\times1}(f_{cls}) \\
p_{cls} &= \text{Sigmoid}(f_{cls})
\end{aligned}
$$

**回归分支：**
$$
\begin{aligned}
f_{reg} &= \text{Conv}_{3\times3}(F) \\
f_{reg} &= \text{Conv}_{1\times1}(f_{reg}) \\
b_{box} &= f_{reg} \quad \text{(预测相对于锚点的偏移量)}
\end{aligned}
$$

**解耦头结构：**

```
┌─────────────────────────────────────────────────────┐
│                  Decoupled Head                     │
├─────────────────────────────────────────────────────┤
│                                                      │
│   Feature ──→ [Conv 3×3 + SiLU] ──→ [Conv 1×1]      │
│      Map                                 ↓           │
│                                     ┌────┴────┐      │
│                                     │ Sigmoid │ → cls
│                                     └─────────┘      │
│                                                      │
│   Feature ──→ [Conv 3×3 + SiLU] ──→ [Conv 1×1] ──→ reg
│      Map                      (无激活函数)           │
│                                                      │
└─────────────────────────────────────────────────────┘
```

### 2. 多尺度检测

YOLOv8 在三个不同尺度的特征图上进行检测，分别负责大、中、小目标的检测：

| 检测层 | 特征图尺寸 | 感受野 | 适用目标 |
|--------|-----------|--------|----------|
| P3 (大目标) | $80\times80$ | 小 | 香烟、打火机（占图像 > 5%） |
| P4 (中目标) | $40\times40$ | 中 | 未穿防护服人员（占图像 2-5%） |
| P5 (小目标) | $20\times20$ | 大 | 安全帽、口罩（占图像 < 2%） |

### 3. 无锚框机制

YOLOv8 舍弃了基于锚框（Anchor-based）的检测范式，采用无锚框（Anchor-free）策略，直接预测目标边界框的四个边界相对于特征图网格点的偏移量。

**预测输出：**

对于特征图上每个位置 $(i, j)$，预测输出为：

$$
o_{i,j} = \left[ d_x, d_y, d_w, d_h, p_{obj}, p_1, p_2, \ldots, p_C \right]
$$

其中：
- $d_x, d_y$：边界框中心相对于网格左上角的偏移量（归一化到 $[0, 1]$）
- $d_w, d_h$：边界框宽高的自然对数：$d_w = \ln(w), d_h = \ln(h)$
- $p_{obj}$：目标置信度（Objectness）
- $p_1, p_2, \ldots, p_C$：各类别的分类概率

**恢复到原始尺度：**

$$
\begin{aligned}
b_x &= \sigma(d_x) \cdot 2^{i-1} + i \cdot 2^{i-1} \\
b_y &= \sigma(d_y) \cdot 2^{i-1} + j \cdot 2^{i-1} \\
b_w &= p_w \cdot 2^{i-1} \\
b_h &= p_h \cdot 2^{i-1} \\
c_x &= i \cdot 2^{i-1} \\
c_y &= j \cdot 2^{i-1}
\end{aligned}
$$

其中 $i$ 为特征金字塔层级，$\sigma$ 为 Sigmoid 函数。

## 3.2.5 损失函数

YOLOv8 的总损失由三部分组成：**分类损失**、**回归损失**和**置信度损失**。

### 1. 分类损失（BCE Loss）

对于分类分支，使用**二元交叉熵损失（Binary Cross Entropy Loss）**：

$$
\mathcal{L}_{cls} = -\frac{1}{N} \sum_{i=1}^{N} \left[ y_i \cdot \log(\hat{y}_i) + (1-y_i) \cdot \log(1-\hat{y}_i) \right]
$$

其中 $y_i$ 为真实标签，$\hat{y}_i$ 为预测概率。

### 2. 回归损失（DFL + CIoU Loss）

YOLOv8 采用**分布焦点损失（Distribution Focal Loss, DFL）**结合 **CIoU Loss** 进行边界框回归。

**CIoU Loss：**

$$
\mathcal{L}_{CIoU} = 1 - IoU + \frac{\rho^2(b, b^{gt})}{c^2} + \alpha v
$$

其中：
- $IoU$：预测框与真实框的交并比
- $\rho^2(b, b^{gt})$：两个框中心点的欧氏距离
- $c$：覆盖两个框的最小闭包区域的对角线长度
- $\alpha$：权重因子
- $v$：长宽比一致性度量：$v = \frac{4}{\pi^2} \left( \arctan\frac{w^{gt}}{h^{gt}} - \arctan\frac{w}{h} \right)^2$

**DFL Loss：**

DFL 将边界框回归问题转化为分类问题，将连续坐标值离散化为多个类别的概率分布：

$$
\mathcal{L}_{DFL} = -\frac{1}{N} \sum_{i=1}^{N} \left[ y_i \cdot \log(p_i) \right]
$$

### 3. 总损失

$$
\mathcal{L}_{total} = \mathcal{L}_{cls} + \lambda_1 \mathcal{L}_{CIoU} + \lambda_2 \mathcal{L}_{DFL} + \lambda_3 \mathcal{L}_{obj}
$$

其中 $\lambda_1, \lambda_2, \lambda_3$ 为各损失项的权重系数（YOLOv8 默认为 $\lambda_1=7.5, \lambda_2=1.5, \lambda_3=1.0$）。

## 3.2.6 本系统的 YOLOv8 适配

针对化工厂安全行为监控的实际需求，本系统对标准 YOLOv8 进行了针对性适配：

### 网络配置

| 参数 | 配置值 | 说明 |
|------|--------|------|
| 输入尺寸 | $640\times640$ | 平衡精度与推理速度 |
| 类别数 | 5（吸烟、动火、无防护装备1、无防护装备2、无防护装备3） | 根据实际违规行为定义 |
| 骨干网络 | YOLOv8s | 在 Jetson Orin Nano 上的性能平衡 |
| 检测尺度 | $80\times80, 40\times40, 20\times20$ | 多尺度覆盖不同大小目标 |

### 轻量化策略

为适应边缘设备的算力限制，本系统采用以下轻量化策略（详见第四章）：

1. **通道剪枝**：移除不重要的卷积通道，减少 30% 计算量
2. **知识蒸馏**：用大型模型指导小型模型训练，保持精度
3. **INT8 量化**：将 FP32 权重转换为 INT8，降低内存占用和推理延迟
4. **TensorRT 加速**：利用 NVIDIA TensorRT 优化推理引擎

## 3.2.7 本章小结

本章详细介绍了 YOLOv8 的网络整体架构及其在本系统中的适配方案。从整体架构来看，YOLOv8 采用经典的 Backbone-Neck-Head 三段式设计，通过 CSPDarknet 骨干网络提取多尺度特征，利用 PANet + FPN 实现双向特征融合，再由解耦检测头完成分类与回归任务。这种设计在保持 YOLO 系列实时检测优势的同时，显著提升了模型精度。

在骨干网络层面，Focus 模块以零计算成本实现了空间下采样与通道扩展；C2f 模块通过 CSPNet 的特征分离与密集连接策略增强了梯度流动；SPPF 模块则以串行化的多尺度池化替代了传统的并行 SPP 结构，在等效感受野下将计算量降低约 50%。这三个模块的协同作用，使骨干网络具备了高效提取多层次语义特征的能力。

颈部网络采用 FPN 自顶向下与 PAN 自底向上的双向特征融合机制，有效解决了深层特征图中小目标信息丢失的问题。FPN 将高层语义信息逐层向下传递，PAN 则将低层细节信息逐层向上聚合，两个方向的特征交互确保了不同尺度目标都能获得丰富的多尺度表征。

检测头的设计是 YOLOv8 的核心创新之一。解耦头将分类与回归任务彻底分离，避免了耦合设计中两个任务相互干扰的问题；无锚框机制摒弃了预定义的锚框模板，使模型对目标尺度和形状变化的适应性更强；三尺度检测（P3/P4/P5）则分别覆盖了大目标、中目标和小目标的不同检测需求。针对化工厂安全监控场景，本章确定了 YOLOv8s 作为基准模型，配合通道剪枝、知识蒸馏、INT8 量化和 TensorRT 加速等轻量化策略，可在 Jetson Orin Nano 边缘设备上实现满足实时性要求的违规行为检测。
